# Dietician Mobile App

A React Native patient app built with Expo. Patients log in with credentials generated by the admin dashboard, view their profile, and see their daily diet plan.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | React Native + Expo SDK ~52 |
| Language | TypeScript |
| Navigation | React Navigation v6 (native-stack) |
| Auth state | Zustand |
| Backend / DB | Firebase Auth + Firestore (same project as dashboard) |
| Auth persistence | `@react-native-async-storage/async-storage` |

---

## Project Structure

```
mobile/
  App.tsx                        # Root — Firebase auth listener, loads user profile
  app.json                       # Expo config
  babel.config.js
  tsconfig.json
  .env                           # Firebase environment variables (EXPO_PUBLIC_ prefix)
  src/
    types/
      index.ts                   # Shared TypeScript interfaces (UserProfile, DietPlan, Meal, …)
    services/
      firebase.ts                # Firebase app init with AsyncStorage persistence
      authService.ts             # signIn (userId or email), signOut
      userService.ts             # Fetch patient profile from Firestore by email
      dietPlanService.ts         # Fetch active diet plan (latest assignedAt)
    store/
      authStore.ts               # Zustand — firebaseUser, userProfile, dietPlan, isLoading
    navigation/
      AppNavigator.tsx           # Auth-gated stack: Login ↔ Home
    screens/
      LoginScreen.tsx            # Patient login screen
      HomeScreen.tsx             # Patient home — profile + today's diet plan
```

## Screens

### Login
- Patient enters their **User ID** (e.g. `priya2026abc`) — no need to type the full email
- The app automatically constructs the Firebase Auth email as `{userId}@dietapp.com`
- Password is the 8-character credential generated by the admin
- Friendly error messages for wrong credentials, too many attempts, etc.

### Home
- **Profile card** — name, age, gender, weight, height, BMI (colour-coded by category), goal, diet preference, allergies, body type
- **Today's meals** — automatically shows the current weekday's meals from the active diet plan
- **Macro summary** — total calories, protein, carbs, fats for the day
- **Meal cards** — meal name, time, food items, per-meal macros, and optional notes
- **Health notes** — conditions and medications if present
- Pull-to-refresh to reload the diet plan
- Logout button

## Authentication Flow

1. Patient opens the app — `onAuthStateChanged` in `App.tsx` checks for a persisted session.
2. While Firebase resolves, a loading spinner is shown.
3. If a session exists, the profile is loaded from Firestore and the Home screen is shown.
4. If not authenticated, the Login screen is shown.
5. On login, `signInWithEmailAndPassword` is called. On success, `AppNavigator` automatically switches to Home.
6. On logout, auth state is cleared and the Login screen is shown.

## Firebase / Firestore

Uses the **same Firebase project** as the admin dashboard (`dietician-app-bd2ff`).

### Collections read by the mobile app

| Collection | Usage |
|---|---|
| `users` | Query `where('userEmail', '==', currentUser.email)` to load the patient profile |
| `users/{id}/dietPlans` | Fetch the most recent plan (`orderBy('assignedAt', 'desc') limit(1)`) |

Patient Firebase Auth email format: `{userId}@dietapp.com`

## Environment Variables

Create a `.env` file in the `mobile/` directory (already included — do not commit to public repos):

```env
EXPO_PUBLIC_FIREBASE_API_KEY=
EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=
EXPO_PUBLIC_FIREBASE_PROJECT_ID=
EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=
EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
EXPO_PUBLIC_FIREBASE_APP_ID=
```

Variables prefixed with `EXPO_PUBLIC_` are automatically available via `process.env.EXPO_PUBLIC_*` in Expo SDK 49+. No extra Babel plugin is needed.

## Setup & Running

```bash
cd mobile
npm install
npx expo start
```

| Key | Action |
|---|---|
| `a` | Open on Android emulator |
| `i` | Open on iOS simulator (macOS only) |
| Scan QR | Open in Expo Go on a physical device |

### Prerequisites
- Node.js 18+
- Expo CLI: `npm install -g expo-cli` (optional — `npx expo` works without it)
- For Android: Android Studio + emulator
- For iOS: Xcode (macOS only)
- For physical device: [Expo Go](https://expo.dev/go) app

## Relationship to the Admin Dashboard

| Concern | Dashboard (`/dashboard`) | Mobile (`/mobile`) |
|---|---|---|
| Creates patients | ✅ | ❌ |
| Assigns diet plans | ✅ | ❌ (read-only) |
| Manages templates | ✅ | ❌ |
| Patient login | ❌ | ✅ |
| View own profile | ❌ | ✅ |
| View today's meals | ❌ | ✅ |

Credentials shown to the admin in the **Credentials tab** of a patient profile are what the patient uses to log in to this app.
